SharpSvn-Daily Update - 1.7004.2064

The SharpSvn project provides frequent builds of the current development tree. These versions are not fully tested, but may contain bugfixes (and new bugs) that are not in released versions. If you are testing these daily builds (Thanks!) and find issues in it, please report those issues with the exact version.

The latest daily build is SharpSvn 1.7004.2064.

The build script also supplies an x64 version of this Win32 build and a package of all binaries with some extras.

You can find older daily releases in the daily folder of documents & files. The previous version of this report is still available in our Subversion Repository.

Published daily builds will be available for at least a month after they are build.

Recent changes

2012-04-11
[r2063] rhuijbensrc/SharpSvn: SvnClientConfiguration.h, SvnEnums.h, SvnClientContext.cpp, SvnClientConfiguration.cpp, SvnClientContext.h
Add an SvnClient.Configuration.KeepAllExtensionsOnConflict property to allow changing the default setting of which extensions to preserve in an easier way. Preserving the extension helps most diff and merge tools detect the right filetype.
2012-03-28
[r2052] rhuijbensrc/SharpSvn: SvnTools.cpp
Optimize SvnTools.GetNormalizedFullPath() for the case where the path is already normalized. This adds another 2.5* performance improvement over r2051. The time is now 2.5 seconds (which used to be 26 seconds).
[r2051] rhuijbensrc: SharpSvn/SvnBase.h, SharpSvn/SvnTools.cpp, SharpSvn.Tests/SvnToolsTests.cs
Improve the performance of checking for invalid paths in SvnTools.GetNormalizedFullPath().
This makes calling GetNormalizedFullPath() take about a fourth of its former time on my testcase (26 sec -> 7 sec for 800,000 fairly long paths)
